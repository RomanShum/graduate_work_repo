# Кино Вместе - Проектная работа

## 🎬 Описание проекта

Проект представляет собой платформу для совместного просмотра фильмов. Пользователи могут создавать комнаты для просмотра фильмов, общаться в чате, синхронизировать воспроизведение видео.

Система построена по микросервисной архитектуре с использованием современных технологий и включает в себя:

- Систему аутентификации и авторизации
- Сервис управления пользовательским контентом
- Систему уведомлений
- Веб-сокет сервер
- Фронтенд приложение

## 📦 Контейнеры в проекте

### 1. Nginx (nginx)
- **Описание**: Обрабатывает входящие HTTP-запросы, разделяет трафик между сервисами и отдает статические файлы

### 2. PostgreSQL (cinema_db)
- **Описание**: Хранит данные пользователей, комнат и другую структурированную информацию
- **Инициализация**: Загружает дамп базы из `configs/database_dump.sql`

### 3. Redis (redis) 
- **Описание**: Используется для хранения сессий пользователей и кэширования данных

### 4. Auth Service (auth_service) 
- **Описание**: Управляет регистрацией, входом пользователей, ролями и правами доступа

### 5. UGC Service (ugc)
- **Описание**: Хранит лайки, избранные фильмы и отзывы пользователей в MongoDB

### 6. MongoDB (mongodb) 📊
- **Описание**: Хранит неструктурированные данные

### 7. Kafka (kafka) 
- **Описание**: Используется для асинхронной передачи событий между сервисами

### 8. Notification Service (notification) 
- **Описание**: Создает и подписывается Подписывается на события в Kafka и отправляет уведомления пользователям по email

### 10. WebSocket Server (ws_app)
- **Описание**: Обеспечивает работу чата, синхронизацию воспроизведения видео и управление комнатами

### 11. MailHog (mailhog)
- **Описание**: Перехватывает и отображает исходящие email-сообщения для тестирования

## ▶️ Запуск проекта

1. Создайте файл `.env` в корне проекта с необходимыми переменными окружения:

2. Запустите проект с помощью Docker Compose:
```bash
docker-compose up --build
```

3. После запуска всех сервисов, откройте веб-интерфейс по адресу `http://localhost`

## 🧪 Запуск скриптов с тестовыми данными

Проект включает несколько скриптов для генерации тестовых данных:

### 1. Создание суперпользователя
```bash
# Войдите в контейнер auth_service
docker exec -it auth_service bash

# Запустите скрипт создания суперпользователя
python create_superuser.py
```
Скрипт запросит логин, пароль, имя и фамилию для создания администратора системы.

### 2. Создание тестовых пользователей
```bash
# В контейнере auth_service
python create_test_users.py --count 20 --base-login user --password password
```
Создает указанное количество тестовых пользователей с префиксом логина и единым паролем.

### 3. Создание тестовых лайков и избранных фильмов
```bash
# В контейнере ugc_service
python create_test_likes.py --users-limit 6 --films-limit 30
```
Генерирует тестовые данные оценок и избранных фильмов для существующих пользователей и фильмов из базы данных.

## 🛠️ Технологический стек

- **Backend**: FastAPI, SQLAlchemy, Beanie, Motor
- **Frontend**: React
- **Базы данных**: PostgreSQL, MongoDB, Redis
- **Очереди**: Kafka
- **Сервер**: Nginx
